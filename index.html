Integration eines kommerziellen Sprachassistenten in eine eigene Audioanwendung

Jiani Qu, Artur Solomonik

Amazon Alexa: Skill Entwicklung

  • Skill: Personalisierte Sprachanwendung
  • Hauseigenes öffentliches Toolkit für Entwicklung (Alexa Interaction Model)
  • Amazon bietet Deployment über Amazon Web Services an
    • Deployment über eigene Server möglich

Amazon Alexa: Interaction Model

  • Benutzeranfragen (Intents) als Interaktionsgrundlage
    • Beispielausdrücke (Sample Utterance)
    • Lexikon für Satzbestandteil eines speziellen Typs (Slots)
      • Vor- oder Benutzerdefiniert
    • Dialog Model für Verständnisprobleme

Amazon Alexa: Interaktionsprozess

  1. Öffnung einer Sitzung
  2. Amazon Echo schickt Stimmaufnahmen an Cloud-Service
  3. Umwandlung der Audioaufnahme in Text
  4. Abgleich mit passender Sample Utterances
    • Ähnlichkeit/Zugehörigkeit zu definierten Slots führt zu Lernprozess
  5. Passende Aktion am Nutzerendpunkt durchgeführt
    • Benutzerdefinierte Antwort: Text-To-Speech Umwandlung der definierten Antwort

RufUs: Funktionalität

  • Idee: Simpler Audiomixer für Alexa
  • Verarbeitung und Wiedergabe mehrerer Audiospuren
  • Hinzufügen / Entfernen von Audiospuren während der Laufzeit einer Datei
  • Schnelle, fließende Interaktion mit Alexa

RufUs: Problemstellung

  • Alexas limitierte Audioausgabe für mehrere Audiospuren ausreizen
  • Möglichkeiten finden mit der aktuellen Audiowiedergabe-Sitzung dynamisch zu interagieren
  • Java-Frameworks für Audioverarbeitung mit Alexa verbinden

RufUs: Limitierungen (Audiowiedergabe)

  • Feste Abtastrate von 16000 Hz
  • Bitrate von 48 kb/s
  • Verwendung von MPEG Version 2 (libmp3lame-Codec)
  • Audiodateien müssen sich auf einem SSL-verschlüsselten, extern zugänglichen Server befinden

RufUs: Limitierungen (Audioverarbeitung)

  • Externes Backend-Modul zwingend notwendig
  • Externer Mischvorgang erzwingt die Interaktion über HTTP-Requests
  • Massives Timeout-Problem bei längeren Mischvorgängen

RufUs: Umsetzung

  • Alexa-Agent nimmt Aktion und Spur vom Nutzer auf
  • HTTP-Request an eine Flask REST-API auf einem SSL verschlüsselten Server
  • Python-Backend mischt die Spuren je nach Request-Methode
  • Alexa fragt nach erfolgreicher Übermittlung die Abgelegte Datei ab und spielt sie ab
